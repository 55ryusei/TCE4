# TCE 4.0.0 セットアップ手順

3.1.0 にクラウドバックアップ機能を追加した版。
**Supabaseのキーを設定するまでは、3.1.0と完全に同じ動作**（クラウド機能はオフ）。

## 仕組み

- 打刻・削除・復元のたびに、全データのコピーをSupabaseへ自動送信（1日1行を上書き、過去日は履歴として残る）
- 電波がないときは端末に貯めて、つながったら自動再送（画面左下に状態バッジが出る）
- `viewer.html` を自宅PCのブラウザで開く → 管理者ログイン → 月ごとの集計・明細を閲覧、JSONダウンロード（アプリの「復元」でそのまま読み込める）

## 初回セットアップ（シフトツールと同じ流れ）

1. https://supabase.com で **新しいプロジェクト**を作る（名前: timecard など。東京リージョン）
   ※シフトツールとは別プロジェクトにする（鍵を分けるため）
2. SQL Editor で `cloud-setup.sql` の中身を実行（Ctrl+Aで空にしてから貼る）
3. Authentication → Users → Add user で管理者アカウントを作成（Auto Confirm ON）
4. Project Settings → API Keys から **Project URL** と **publishableキー** をコピー
5. `index.html` と `viewer.html` の先頭付近にある `CLOUD_URL` / `CLOUD_KEY` に貼る
   （Claudeにキーを渡せば書き込んでもらえる）
6. iPadへは今まで通りの方法で配置（sw.jsのキャッシュ名はv4に更新済みなので、配置すれば新版に切り替わる）

## 月々の運用（4.0.0からの新ルール）

- **iPadは打刻するだけ。回収・バックアップ・全データ削除は不要**
- 従業員の一覧には「今の期間（11日〜翌月10日）」の記録だけが表示される
  （古いデータは消えていない。端末内に残り、クラウドにも保存されている）
- 給与計算は自宅PCで `viewer.html` を開いて、期間を選んで集計を見る
- バックアップ・復元・全削除ボタンは非常用として残してあるが、通常は使わない

## 名前のルール（従業員への周知事項）

- **名前は一度決めたら変えない**（途中で変えると給与集計がその人だけ2つに分裂する）
- 60日間使われていない名前はプルダウンから自動で消える（データは消えない）
- 休職などから復帰した人は、**前と同じ名前**を手入力して打刻すれば過去の記録とつながり、プルダウンにも復活する

## セキュリティ

- iPadに埋め込むキーは「送信専用」。万一漏れても読み出しはできない
- 過去の履歴（前日より前）は上書き不可 → データ破損やいたずらの保険
- 閲覧はSupabaseの管理者ログインが必要
